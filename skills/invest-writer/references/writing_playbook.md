# 投资长文写作手册

这份手册沉淀的是一种更适合公众号和长帖阅读的投资写法，不是传统研究报告写法。

## 零、选题策略：写什么比怎么写更重要

选题本身自带流量系数。经过多篇实际文章的阅读量复盘，形成以下判断：

### 高传播选题（自带流量）

这几类选题天然有更高的传播上限：

**1. 新闻驱动型发现**
- 政策文件刚发布 → 第一时间解读出投资含义
- 财报刚公布 → 发现市场没注意到的异常数据
- 突发行业事件 → 快速给出分析框架

核心：你不是在复述新闻，而是在新闻上叠加一层判断——"这意味着什么会改变"。

案例："央企利润上缴翻倍"这篇文章的核心竞争力：财政部文件刚出，你第一个解读出它对高股息央企和成长型央企的不同影响。

**2. 市场分歧型站队**
- 一个话题市场上有两派激烈争论 → 给出明确立场和推理
- "到底是衰退前兆还是转型阵痛？"——直接回答

案例："中国不是下一个日本"——直接对抗市场上最流行的悲观叙事。

**3. 被验证的长期跟踪**
- 一个标的你跟踪了很久，有阶段性复盘可写
- 判断被验证或需要修正

### 低传播选题（除非有独特增量，否则慎选）

**1. 书的读后感/框架转述**
- 除非书本身极新且有争议，否则天然传播力弱
- 读者感知："你读了本书，跟我有什么关系？"
- 例外：如果能提炼出一个基于书的原创判断框架，且该框架对当前热门话题有解释力

**2. 已过时效的数据分析**
- 去年的消费率、去年的GDP结构——数据已经不新鲜
- 除非你能用旧数据回答一个当前热门问题

**3. 纯概念/方法论文章**
- 没有具体案例锚定的"投资方法论""行业分析框架"
- 读者不会转发框架，只会转发洞察

### 决策口诀

```
优先写：被新闻触发、被好奇心驱动的"发现"
其次写：被长期跟踪验证的"判断"
尽量不写：被阅读激发的"转述"
```

## 一、文章的核心任务

不是把资料写全，而是把一个关键判断写透：

- 这次下跌到底是趋势坏了，还是短期扰动？
- 市场为什么会这样定价？
- 真正重要的变量到底是什么？

如果你没有先找到这个问题，后面结构再完整，文章也容易散。

## 二、九条可复用规则

### 1. 开头即钩子

开头不要先预告结构，直接抛问题。

推荐公式：

```text
现象陈述 -> 反直觉问题 -> 说明 A 和 B 差别很大
```

示例：

- 一般写法：最近公司发布了年报，下面从几个方面展开分析。
- 更好的写法：股价跌了不少，但我看完年报后真正想问的是，这到底是趋势变差，还是短期波动？

### 2. 标题必须有数字或冲突

好的标题至少满足以下两条中的一条：

- **数字暴力**：用"翻倍"不用"提升"、用"腰斩"不用"下降"
- **认知冲突**：把两个矛盾的东西放一起，如"静悄悄" vs "价值重估"

组合效果排序：

```
数字 + 冲突 > 纯冲突 > 纯数字 > 概念陈述 > 分类标签
```

正面案例：

- "央企利润上缴翻倍——一场静悄悄的价值重估" ← 数字（翻倍）+ 冲突（静悄悄 vs 重大）
- "中国不是下一个日本——答案不在宏观数据里，在产业生态里" ← 冲突（对抗流行叙事）+ 解法预告

反面案例：

- "中国经济成人礼——从追赶到成熟的十年" ← 只有概念，没有数字也没有冲突
- "海尔智家：存量市场中的价值锚点" ← 像研报标题

### 3. 结构扁平化

每一节都独立成篇，标题本身就是一句问题或结论。

不要写：

- 行业分析
- 企业分析
- 财务分析

改成：

- 问题到底出在哪？
- 为什么这次会跌这么多？
- 行业其实已经变了

### 4. 数据只留最有说服力的

默认只保留 3-5 个核心数据。

保留什么：

- 能解释当前争议的数据
- 能区分短期和长期的数据
- 能支撑估值判断的数据

删除什么：

- 与主线关系不大的行业背景数据
- 为了显得专业而罗列的来源
- 一眼看不出意义的细碎指标

### 5. 论证必须层层递进

推荐逻辑链：

```text
现象 -> 原因 -> 性质判断 -> 更大的背景 -> 机会 -> 估值 -> 结论 -> 个人行动
```

这条链最重要的作用，是让每一段都在回答读者的下一个问题。

### 6. 控制节奏——每 300 字一个呼吸点

手机阅读场景下，读者边滑边做"是否继续读"的决策。连续三段以上长文字 = 高概率流失。

**规则**：

- 每个自然段 2-4 句，不超过 5 句
- 每 200-300 字出现一个呼吸点：段落间空行、分割线或新小标题
- 核心观点用独立短句呈现
- 小标题之间保持 2-4 段正文量

**典型节奏示意**：

```
短段落（2-3句）

短段落（2-3句）

---

### 问题式小标题

短段落

核心句独立成段。

短段落

---
```

**对比**：央企文章之所以在小红书炸了，一个重要原因是它天然适配"短段落 + 高频呼吸点"的节奏。而"文章1"和"文章2"存在大量连续长段落，在手机端阅读体验明显更重。

### 7. 用分割线和小标题控制节奏

推荐格式：

```markdown
---

### 一、先看数据：问题出在哪？

[内容]
```

好处：

- 手机阅读更轻松
- 每节天然形成停顿
- 观点之间不容易混在一起

### 8. 结尾写成"自白"，不要写成"建议"

结尾最好包含四个元素：

```text
我的犹豫 -> 我想明白的点 -> 我的操作方式 -> 更高一层的反思
```

读者更想看的是你的判断过程，而不是一句"值得长期关注"。

### 9. 删除"正确的废话"

典型无效句子：

- 根据某某数据……
- 这是一个本质的变化……
- 为什么这么判断？因为前文已经说过的内容又重复一遍

删减原则：

- 不重复
- 不解释常识
- 不为了专业感而拉长

## 三、身份定位：发现者 > 翻译者

经过多篇文章的传播效果对比，发现一个规律：**读者愿意传播的不是"你读了什么"，而是"你发现了什么"。**

三种身份及其传播力：

| 身份 | 读者感知 | 传播力 | 典型文章 |
|------|---------|--------|---------|
| **发现者** | "这个人看到了我没想到的东西" | 高 | 央企利润上缴翻倍 |
| **解读者** | "这个人帮我理解了复杂的事" | 中 | 中国不是下一个日本、AI时代哪些公司值得关注 |
| **翻译者** | "这个人把别人说的又说了一遍" | 低 | 文章1/2/3（书的内容转述）、HALO文章 |

**如何从翻译者变成发现者**：

- 素材是书 → 不写"这本书讲了什么"，写"这本书让我想明白了什么困扰我的问题"
- 素材是报告 → 不写"报告指出"，写"报告里有一个判断和我之前的想法相反"
- 素材是新闻 → 不只复述新闻，叠加一问"这意味着什么会改变"

**检验标准**：文中"我注意到""我发现""我反复想"类表达，是否多于"某某说""报告指出""根据数据显示"。

## 四、一句话可转述：观点的传播单位

传播力最强的文章都有一个共同特征：核心观点可以用一句话转述。

检验方法：假设读者在饭局上聊到这篇文章，他会怎么说？

✅ 可转述的：

- "央企利润上缴翻倍了，以后高股息央企会越来越像债券，成长型央企现金流会吃紧。"
- "判断中国经济不能看GDP增速，要看它持续创造新产业的能力。"
- "AI时代真正不怕AI的公司，是那些拥有重资产、强品牌、深渠道的公司。"

❌ 不可转述的：

- "中国经济正在经历从追赶到成熟的转型。" ← 太模糊，不知道转述什么
- "消费率只有四成，有多层原因。" ← 有信息没态度

写作时把这个一句话版本先写出来，然后围绕它展开全文。如果写不出来，说明核心观点还不够锐。

## 五、平台策略：同一篇文章，两个平台两种命运

同一篇内容在小红书（10万+）和公众号（低阅读）的差距，揭示了平台逻辑的根本差异。

### 微信公众号：订阅驱动的信任积累

**分发逻辑**：粉丝订阅 + 朋友圈转发。没有基础粉丝，好文章也难破圈。

**适合什么**：
- 1500-3500 字的深度长文
- 需要时间建立信任的内容（你是认真在做研究的人）
- 系列化的跟踪分析

**转发动力**：
- 观点够锐，帮读者说出了他想说但说不清的话
- 信息增量大，读者觉得"看了这篇省了我自己研究的时间"
- 框架清晰，读者可以用它去看别的事情

**冷启动策略**：
- 有几千基础粉丝之前，公众号天然存在阅读天花板
- 需要持续输出到"临界质量"——某篇转发多了，才能触发公众号内的二次传播

### 小红书：算法驱动的兴趣匹配

**分发逻辑**：兴趣推荐算法。标题关键词决定初始推荐对象，互动率决定流量池跃迁。

**适合什么**：
- 800-2000 字轻量化表达
- 短段落、多呼吸点的节奏
- "看完能跟人聊两句"的轻量认知增量

**标题策略**：
- 必须包含高频搜索词：政策术语、行业关键词、热点争议词
- 数字和冲突是点击率的核心驱动力
- 小红书标题有字数限制，但关键词密度比长度重要

**内容策略**：
- 节奏比深度重要：宁可观点浅一点，不能段落长一点
- 用户在做"滑屏决策"，每 2-3 秒给一个继续读的理由
- 评论区互动有助于触发算法二次推荐

### 一鱼多吃的实操方法

一篇长文写完，做两个版本：

1. **公众号版**：完整结构，1500-3500 字，深度完整
2. **小红书版**：砍到 800-1500 字，标题加数字和热搜词，段落进一步切短，核心观点前置

不要试图一个版本适配所有平台。这大概是这次复盘最直接的操作性结论。

## 六、默认成文模板

```markdown
# 标题

[现象 + 问题]

---

### 一、先看数据：问题出在哪？

---

### 二、为什么会这样？

---

### 三、这些影响是长期的吗？

---

### 四、更大的背景是什么？

---

### 五、未来的机会在哪里？

---

### 六、估值怎么看？

---

### 七、最后说结论

---

### 八、说点更真实的
```

## 七、什么时候可以偏离模板

这些情况可以适当改结构：

- 行业专题：把"公司问题"换成"行业正在发生什么变化"
- 多公司对比：每节围绕一个对比维度展开
- 方法论文章：减少估值部分，强化认知和框架部分

但就算偏离，也尽量保留这三点：

- 问题驱动开头
- 扁平化小节
- 第一人称结尾

## 八、最后一道检查

成稿出来后，快速问自己：

1. 如果删掉标题，读者还能顺着问题链读下去吗？
2. 每一节是不是都比上一节更靠近结论？
3. 结尾是不是从"判断"走到了"行动"？
4. 这篇文章像不像一个真实投资者写的，而不是一个会写字的分析模板？
5. 核心观点能不能用一句话转述？
6. 有没有哪个段落连续超过 5 行？有的话，切开。
